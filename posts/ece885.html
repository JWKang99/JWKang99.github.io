一、ArrayList

1.概述

因为ArrayList是基于数组实现的，所以支持快速随机访问。RandomAccess接口标识着该类支持快速随机访问。继承实现关系：实现了四个接口一个抽象类。

3.增加元素

  • boolean add(E e)：添加一个元素到列表末尾，
  • boolean add(int index,E e)：在指定位置添加元素
  • boolean addAll(Collection<? extends E> c)：添加一个集合到元素的末尾
  • void ensureCapacityInternal(int minCapacity)：确保列表中含有minCapacity的最小容量，扩容是会把原数组整个复制到新数组，代价很高，故最好初始化时指定大概容量。
  • void grow(int minCapacity)：容量不够时进行扩容为原来的1.5倍。
java

4.删除元素

需要调用System.arraycopy()将后边的元素都往前移动，时间复杂度为0(N)。

  • E remove(int index)：删除指定位置的元素
  • boolean remove(Object o)：删除第一个出现的o元素
  • void removeRange(int fromIndex, int toIndex)：范围删除，把后边的往前移动，多余位置置为null
  • boolean removeAll(Collection<?> c)：删除两者个交集，调用了batchRemove()用两个位置指针标记，多于位置置为null
java

5.更改元素

  • E set(int index, E element)：更改元素内容，返回旧值
  • boolean retainAll(Collection<?> c)：只保留两个集合的交集。
  • void trimToSize()：把列表的容量改为size
java

ArrayList 基于数组实现，并且具有动态扩容特性，因此保存元素的数组不一定都会被使用，那么就没必要全部进行序列化。

modCount 用来记录 ArrayList 结构发生变化的次数。结构发生变化是指添加或者删除至少一个元素的所有操作，或者是调整内部数组的大小，仅仅只是设置元素的值不算结构发生变化。

3.与ArrayList的比较

  • Vector 是同步的，因此开销就比 ArrayList 要大，访问速度更慢。最好使用 ArrayList 而不是 Vector，因为同步操作完全可以由程序员自己来控制；

三、CopyOnWriteArrayList

CopyOnWrite意为写入时复制，核心思想时：如果有多个线程要求读取同样的资源时，这些线程会共同获取指向该资源相同的指针，当有某个线程要修改该资源时都会创建该资源的新副本，在副本上进行修改，写完之后再用副本替换原来的数据，原来的数据内存就会被回收。

java

1.读写分离

写操作在一个复制的数组上进行，读操作还是在原数组中进行，读写分离，互不影响。

写操作需要加锁，防止并发写入时导致数据丢失。

写操作结束之后需要把原始数据指向新的复制数组。

java
java

2.适用场景

CopyOnWriteArrayList 在写操作的同时允许读操作，大大提高了读操作的性能，因此很适合读多写少的应用场景。

但是 CopyOnWriteArrayList 有其缺陷：

  • 内存占用：在写操作时需要复制一个新的数组，使得内存占用为原来的两倍左右；
  • 数据不一致：读操作不能读取实时性的数据，因为部分写操作的数据还未同步到读数组中。

所以 CopyOnWriteArrayList 不适合内存敏感以及对实时性要求很高的场景。

1.概述

LinkedList基于双向链表实现，实现四个接口和一个抽象类，其中AbstractSequentialList继承自AbstractList，该类为了减少LinkedList这种顺序存储的类的代码复杂度而抽象出来的。

同时实现了Deque接口，有队列的各种特性。

3.添加元素和删除元素

  • boolean add(E e)：和ArrayList类似在列表末尾添加元素，依赖于linkLast方法
  • void linkLast(E e)：在末尾添加一个元素，同样还有删除最后一个结点的方法
  • void linkFirst(E e)：在头部添加一个元素，同样还有删除第一个结点的方法
  • void linkBefore(E e, Node<E> succ)：在succ之前添加一个元素
  • boolean add(E e)：
java

4.与ArrayList对比

ArrayList 基于动态数组实现，LinkedList 基于双向链表实现。ArrayList 和 LinkedList 的区别可以归结为数组和链表的区别：

  • 数组支持随机访问，但插入删除的代价很高，需要移动大量元素；
  • 链表不支持随机访问，但插入删除只需要改变指针。
